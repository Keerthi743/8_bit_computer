# 8-Bit Computer Architecture using Verilog HDL

## Overview

This project implements a simple **8-bit computer architecture** using **Verilog HDL**.  
The aim of this project is to design and simulate a basic CPU system with a datapath and control unit, similar to a simple SAP-style processor.

The processor is designed with separate hardware modules and controlled using clock cycles and control signals generated by the Control Unit.


## Project Features

- 8-bit data bus architecture
- Modular RTL design using Verilog HDL
- Separate datapath and control unit
- Instruction fetch and execute cycle
- Memory-based instruction execution
- ALU operations
- Accumulator-based processing
- Flag generation
- Input and output interfacing
  

## Architecture

The computer consists of the following modules:

Datapath Modules

- **Program Counter (PC)**
  - Stores address of the next instruction

- **Memory Address Register (MAR)**
  - Holds the address for memory access

- **Memory Unit**
  - Stores instructions and data

- **Instruction Register (IR)**
  - Stores the currently fetched instruction

- **Accumulator (ACC)**
  - Main working register for arithmetic operations

- **B Register**
  - Temporary register for ALU operations

- **Arithmetic Logic Unit (ALU)**
  - Performs arithmetic and logical operations

- **Flag Register**
  - Stores status flags such as:
    - Zero flag
    - Carry flag

- **Bus**
  - Common data transfer path between modules


Control Modules

- **Control Unit**
  - Generates control signals based on:
    - Clock cycle
    - Instruction opcode
    - Processor state

- **Instruction Decoder**
  - Separates instruction opcode and address fields

- **Clock Generator**
  - Provides timing sequence for processor operations


## Instruction Flow

The processor works using the following cycle:

1. Fetch Cycle
The instruction is fetched from memory.

2. Decode Cycle
The Control Unit decodes the instruction.

3. Execute Cycle
The required operation is performed.
